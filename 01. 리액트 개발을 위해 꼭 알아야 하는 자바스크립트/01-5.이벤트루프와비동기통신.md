= 기본적으로 한 번에 하나의 작업만 동기 방식 처리 가능

# 싱글 스레드 자바스크립트

스레드란?

-   프로세스(process): 프로그램을 실행하는 단위
-   스레드(thread): 프로세스 내에 더 작은 실행 단위. 하나의 프로세스는 여러 개의 스레드를 만들 수 있고, 스레드끼리는 '메모리를 공유할 수 있다'

하지만 Javascript는 싱글 스레드네? 왜?

멀티 스레드는 여러 장점이 있지만 내부적으로 처리가 복잡하다는 단점 존재(원래 Javascript는 HTML을 보조하는 역할 정도였기에)

-   동시에 같은 자원(공유 메모리)에 접근하는 것 때문에 생기는 '동시성 문제'
-   프로세스는 격리되어 있지만, 스레드의 경우 어떤 스레드가 문제가 생겼을 때 다른 스레드에도 문제가 발생할 수 있다.

Javascript의 특징([webWorker](https://developer.mozilla.org/ko/docs/Web/API/Worker) 혹은 [worker](https://nodejs.org/api/worker_threads.html)는 매우 최근에 나왔으므로 논외)

-   C언어 혹은 다른 프로그래밍 언어와 다르게 실행 중인 함수를 시스템이 임의로 멈추고 '다르 스레드' 코드 먼저 실행 불가
    -   "Run-to-completion": 하나의 코드가 실행하는 데 오래 걸리면 뒤 이은 코드가 실행되지 않는 Javascript의 특징
-   동시성 고민 X BUT 웹페이지에서는 단점이 될 수 있음: 웹페이지가 멈춘 것 같은 느낌..!
